# 开发过程记录

> 这份文档按时间顺序记录了用 Claude Code 制作这套 slides 的完整过程，包括每一步做了什么、为什么这么做、踩了什么坑。目的是让你复现这个工作流时有据可依。

---

## Session 1：项目初始化与风格确立（03-16 凌晨，~30min）

### 做了什么

1. **`/init` 创建 CLAUDE.md**：让 Claude Code 扫描项目目录，自动生成初始版 CLAUDE.md
2. **创建 GitHub 仓库并推送**：一句 "创建一个 github 仓库并推送"
3. **安装 frontend-slides skill**：给了一个 GitHub 链接，Claude Code 自动下载并安装到 `.claude/skills/`
4. **提供风格规格生成第一版**：给了一份 YAML 格式的设计规格（"Refined Minimal Portfolio"日式极简风格），Claude Code 一次性生成了 22 页 HTML slides

### 关键决策

- **用 YAML 描述风格**，而不是口头说"好看一点"。YAML 里定义了：配色方案、字体组合、布局模式（split/grid/timeline/chat 等）、明暗交替节奏。这给了 Claude Code 足够具体的视觉约束。
- **一次生成全部骨架**，而不是一页一页来。先有完整结构，再逐页打磨，效率更高。

### 产出
- 22 页 HTML slides 初版（日式极简，黑白交替）
- 基础导航（键盘 + 进度条）

## Session 2：视觉调优——从参考到定稿（03-16 晚，~1h）

### 做了什么

这个 session 的核心是 **确定最终视觉风格**，经历了 6 轮迭代：

1. **加机构 logo**："页面底部加上 CDI 的 logo" → Claude 用 JS 注入到所有页面
2. **加作者信息**："首页加上作者信息：葛亚特 博士生 数字创新中心"
3. **加目录导航**："左下角添加一个 button，点击后弹出目录" → 汉堡菜单 + slide-up 面板
4. **全黑背景**："全部采用黑色背景" → 去掉明暗交替，统一暗色主题
5. **参考 Keynote 配色**：丢了一个之前做过的 `.key` 文件路径 → Claude 解压 Keynote，提取色板和纹理
6. **截图修正**："背景太灰了，深一点。加上科技蓝做装饰色" → 附了一张浏览器截图

### 关键经验

> **确定风格的最有效方式：给参考 + 截图反馈**

- 光说"好看"没用，给具体的视觉参考文件（Keynote/Figma/网页截图）效果远好于文字描述
- 第一版风格几乎不可能一步到位，但 2-3 轮截图反馈就能快速收敛
- Claude Code 能直接解压 Keynote 文件并分析其中的配色和设计元素

### 最终确定的视觉语言
- 背景 `#0A0A0A`（近纯黑）+ SVG 噪点纹理
- 强调色 `#00E8C6`（青绿/科技蓝）
- DM Sans + Noto Sans SC + DM Mono 字体组合
- 柔和的光晕装饰元素

## Session 3：封面粒子动画（03-16 晚 ~ 03-17 凌晨，穿插在主开发中）

### 做了什么

用 p5.js 把 OpenClaw 的龙虾 logo 做成粒子效果作为封面背景，经历了 **约 10 轮微调**：

1. 给 `assets/claw.png`，要求用粒子渲染 + 鼠标涟漪交互
2. "改成纯色粒子" → "还是恢复原色"
3. "涟漪效果太强了" → 降到 1/3
4. "再放大 1.5 倍" → "再加透明度"
5. "没有鼠标操作时也要有随机涟漪" → "更频繁但更弱" → "同时触发多个"
6. "文字颜色太灰了" → "封面文字再白一点"
7. 后来又在另一个 session 中做了更炫酷的版本（入场聚合动画 + 星座连线 + 磁力排斥）

### 踩的坑

- **CORS 问题**：`file://` 协议下 p5.js 的 `loadImage()` 被浏览器拦截。最终解决方案是把龙虾图片转 base64 内联
- **base64 图片污染上下文**：把大图的 base64 数据传进对话后，Claude API 报 "Could not process image" 错误，整个 session 卡死。**教训：大的二进制数据不要传进对话，用文件路径引用**

### 关键经验

> **视觉动效的调优是最耗轮次的环节**

粒子效果的参数（密度、力度、透明度、频率）很难一步到位，需要 "看效果 → 调参数" 的快速循环。好在每次修改都很小（改个数字），Claude Code 执行很快。

## Session 4：Part 1 内容——概念图表制作（03-16 22:00 ~ 03-17 00:45，~16 commits）

### 做了什么

这是 **内容制作的开始**。Part 1 讲 LLM → Chatbot → Agent → OpenClaw 的演进，每一步都需要一张概念图。

1. **描述四张图的概念**："Part 1 四页分别画 LLM/Chatbot/Agent/OpenClaw 的架构图，四张图要有视觉连续性"
   → Claude 用 inline SVG 画了四张动画图表
2. **要求加图标**：让 Claude 搜索可用的图标库 → 推荐了 Lucide Icons → 加入 SVG path 图标
3. **OpenClaw 架构图反复迭代**（5-6 次）：
   - 第 1 版：右侧线条连接 → 太杂乱
   - 第 2 版：orbit 环绕布局 → 太抽象
   - 第 3 版：加了 Gateway 和 Cron → 位置不对
   - 第 4 版：用户提供核心比喻 "Agent = 给了工具箱，OpenClaw = 给了车库" → 加入对比
   - 第 5 版：内容溢出 → 压缩布局
   - 最终版：竖向紧凑堆叠
4. **Agent Loop 图**：给了 OpenClaw 文档 URL → Claude 读取后画了 pipeline 图 → 嫌乱 → 用 `/frontend-design` skill 重新设计

### 关键经验

> **描述概念比描述布局更有效**

- 与其说"左边放三个方块，右边放一条线连过去"，不如说"核心比喻是：Agent 是给了一个工具箱，OpenClaw 是给了一个车库"——Claude 会自己找合适的视觉表达
- 但布局细节的修正还是需要你来指挥："Gateway 应该在右边" "这几个模块应该环绕中心，不是从右边连线过来"
- 复杂概念图 3-5 轮迭代是正常的，不要期望一步到位

> **给 URL 让 Claude 自己查资料**

不需要自己读完文档再复述，直接给链接让 Claude 去读。本项目中 Claude 读取了 OpenClaw 的 agent-loop、agent-workspace、context 等多个文档页面。

## Session 5：Part 2 内容——大量概念输入（03-17 00:51 ~ 02:44，~34 commits）

### 做了什么

这是 **最密集的开发阶段**，Part 2 的每个章节都涉及大量概念输入和反复迭代：

#### 2.1 主动交互
- 重命名 + 重新设计对比布局（传统 vs Agent）
- 提供两个产品 URL（docko.app、looirobot.com），要求作为桌面宠物案例
- **关键指令**："不要自己画插图，去网站上抓真实的 favicon 和截图"
- 提供专利文件路径（.docx），让 Claude 读取并做成 slide

#### 2.2 长期交互与自适应学习
- 用户口述了三页的内容框架：什么是长期人机交互 → 记忆分类 → 自适应学习的设计机会
- **方向性修正**："角度不对，不应该从 HRI 挑战出发，应该从 Agent 视角讲具身性——大脑有了，身体是什么？"
- Claude 把原内容删了重写 → 用户说原来的也要保留 → 拆成两页

#### 2.3 终端用户编程
- 用户提供了完整的概念框架：个性化光谱（用户自定义 ↔ 系统自学习），2×2 表格（传统/Agent × Design Time/Runtime），"调教" Agent 的概念
- Claude 搜索了学术文献（Horvitz 1999、Findlater & McGrenere）补充内容

#### 2.4 多端交互
- 用户纠正了"范式转变"的说法：不要那么夸张，改成"四个角度"
- 提供了总结的核心洞见："我们不是在设计静态界面，而是在设计人和 Agent 的沟通方式"

#### 学术案例
- Cocobo、GenFaceUI：要求用真实论文 figure，不够就多加一页
- OpenClaw 生态系统（OpenClaw-RL、MetaClaw、RoboClaw）：给名字让 Claude 去搜索

#### 反复调整 slide 顺序
- 至少 5 次 swap 操作
- 典型案例："六大核心组件"和"OpenClaw 介绍"谁先谁后来回调了 3 次

### 关键经验

> **口述概念框架 + Claude 执行 = 最高效的协作模式**

用户用自然语言（很多是语音转文字）描述了每个章节的核心论点和结构，Claude 负责转化为 HTML 和视觉呈现。用户不需要写任何代码。

> **用真实素材，不要让 AI 画**

- 论文 figure、产品截图、专利文档——这些真实素材比 Claude 画的 SVG 示意图有说服力得多
- 本项目中所有学术案例页最终都替换为了真实图片

> **CLAUDE.md 需要在开发中持续更新**

这个 session 中因为反复遇到内容溢出，专门更新了 CLAUDE.md，加入了"每页可用高度只有 500-550px"的约束。之后新生成的页面就不再溢出了。

> **叙事顺序需要从听众视角反复推敲**

技术人员的逻辑顺序（先组件再整体）和听众的理解顺序（先整体再组件）往往相反。Swap 操作很便宜，不要怕调。

## Session 6：真实内容填充——专利与 Temi 机器人（03-17 02:00 ~ 02:44）

### 做了什么

1. **读取专利文档**：给了 `.docx` 文件路径，Claude 用 pandoc 读取，做成了两页 slide（宏观服务时间线 + 微观主动交互决策流程）
2. **第一次修正**：Claude 把内容写到了 Markdown 文件而不是 HTML → 用户纠正 "你搞错了，要改的是网页不是 MD"
3. **参考资料加超链接**：一句话让 Claude 把所有引用变成可点击的链接
4. **Temi 机器人实例**：指向本地 `.openclaw` 工作空间目录，让 Claude 分析 SOUL.md、AGENTS.md、Skills、对话记录，做成三页 slide：
   - 工作空间现状（已配置了什么）
   - 调教过程时间线（一下午 7 个训练节点）
   - 目标、现实与挑战

### 关键经验

> **指向本地目录让 Claude 分析，比自己复述高效得多**

Claude 直接读取了 `.openclaw/` 目录下的所有配置文件和对话记录，自动提取了关键信息并可视化。你不需要先整理好再告诉它。

> **Claude 偶尔会搞错操作目标**

它可能改了 Markdown 文件而不是 HTML。明确指出"改的是 presentation.html 这个网页"就好。

## Session 7-8：离线化与部署（03-17 09:27 ~ 03-18）

### 做了什么

1. **离线打包**：问 Claude "能不能不开服务器直接双击打开" → 它提供了两个方案，选了"内联 p5.js + base64 图片" → 文件增大约 1MB 但实现完全离线
2. **CORS 修复**：内联后 `file://` 下的 `history.replaceState` 也报错 → 包了 try/catch + hash fallback
3. **Favicon**：加了龙虾 emoji 作为 inline SVG favicon
4. **GitHub Pages**：让 Claude 开启 → 发现 404 → 因为文件叫 `presentation.html` 不叫 `index.html` → 加了重定向

### 关键经验

> **单文件 HTML 的离线化就是把所有外部依赖内联**

p5.js（~800KB）+ 图片 base64 + Google Fonts fallback。文件大了，但换来的是双击即用，在教室没网的情况下很有价值。

## Session 9：模板复用——做另一套完全不同的 Slides（03-20）

### 做了什么

用同一个 HTML 模板，把内容全部替换成了 **Formative Study（形成性研究）** 主题的 slides：

1. 指定新的内容结构：定义与目标 → 类型分类 → 方法总结 → GenComUI 案例
2. 让 Claude 从 Obsidian 笔记中查找 formative study 的资料
3. 替换封面粒子效果（龙虾 → 抽象研究网络节点）
4. 从论文 LaTeX 项目中找到真实 figure 并集成

### 关键经验

> **好的模板是可以复用的**

因为所有视觉规范都在 CLAUDE.md 和 CSS 变量里，换一套内容只需要替换 `<section>` 内的 HTML，整体风格自动保持一致。`frontend-slides` skill 也不需要重新配置。

## 可复用的 Checklist

如果你也想用 Claude Code 做 slides，按这个顺序来：

- [ ] 用 Markdown 写好内容大纲（不需要完美，有结构就行）
- [ ] 安装 `frontend-slides` skill（或自己写一个 slides 制作的 skill）
- [ ] 提供视觉参考（Keynote/Figma/网页截图/YAML 风格规格）
- [ ] 一次性生成全部页面的骨架
- [ ] 让 Claude 把设计规范写入 CLAUDE.md
- [ ] 逐页细化：概念描述 → Claude 实现 → 浏览器预览 → 截图反馈
- [ ] 提供真实素材：论文 figure、产品截图、文档 URL、本地文件路径
- [ ] 发现重复问题时更新 CLAUDE.md（如高度约束、命名规范）
- [ ] 每次修改立即 commit
- [ ] 最后处理离线化和部署
