01 Architecture

六大核心组件

01

Gateway

消息路由控制面，连接 20+ 通讯平台。Hub-and-spoke 架构，Channel Adapter 归一化消息格式。

02

Brain

基于 ReAct 循环编排 LLM 调用，支持 Claude、GPT、DeepSeek 等多模型。

03

Memory

双层记忆：短期（daily logs）+ 长期（Markdown 文件）。刻意简洁，无向量数据库。

04

Skills

插件能力系统，YAML frontmatter 定义。社区通过 ClawHub 共享 1700+ 技能。

05

Heartbeat

周期性触发（如每30分钟），无需用户输入。Agent 主动性的关键机制。

06

Cron

定时任务调度，支持 cron 表达式。可设定每日摘要、定期检查等自动化流程。

02 Proactive · Detail

案例续：机器人何时该主动、何时该等待？

FINE-GRAINED PROACTIVE INTERACTION DECISION

预设服务 → 预启动响应

调整位姿、屏幕朝向
安静就位，等用户来

预测服务 → 建议响应

屏幕展示个性化建议
主动引导用户决定是否启动

DESIGN INSIGHT

确定性高 → 默默就位
不确定 → 先建议再行动
主动 ≠ 打扰，关键在分寸

02 Long-term

2.2 长期交互与自适应学习

LONG-TERM HUMAN-AI INTERACTION

WHAT IS LONG-TERM?

HRI 研究定义：与同一用户3+ 次交互，跨 3+ 天
— ACM THRI, 2025; Leite et al., 2013

标志性研究：Kanda et al. 小学部署社交机器人 2 个月，
将机器人视为"朋友"而非"工具"时，交互持续更久

KEY CHARACTERISTICS

用户识别与记忆 — 认出你，记住对话

行为适应 — 根据历史调整风格

关系发展 — 从好奇→信任→伙伴

融入日常 — 成为"正常成员"

核心挑战：新鲜感衰退 — 初始兴奋消退后如何维持参与？
de Graaf et al. 六阶段接受模型：期望 → 接触 → 采用 → 适应 → 整合 → 认同

02 Challenges

Agent 时代的
设计挑战

以上是 Agent 架构带来的创新机会，以下是设计者需要解决的问题

02 Challenges
Challenge 01

透明性与控制权

OpenClaw 实际上是个黑盒：权限极高，消耗大量 token，后台做了很多用户看不到的事情。

Agent 在做什么？做了多少？做对了没有？
如何设计 Agent 行为的可审计性？

Challenge 02

信任校准

用户如何建立对 Agent 的合理信任？

过度信任 → 依赖 Agent，出错不自知
过度怀疑 → 反复确认，Agent 形同虚设

如何帮助用户形成准确的心智模型？

02 Challenges
Challenge 03

错误恢复与问责

Agent 自主执行了错误操作怎么办？发错邮件、删错文件、回复不当。

如何设计撤销/回滚机制？
出了问题"谁负责"？

Challenge 04

隐私与记忆边界

Agent 记住了太多个人信息怎么办？记忆有时效性，过时的记忆可能导致错误判断。

如何设计记忆衰减机制？
多用户场景中记忆的边界在哪里？

Challenge 05

社交场景中的 Agent

Agent 出现在群聊中时，它是什么角色？是某人的私人助手还是群组的公共助手？

Agent 在多人交互中的身份、权限、发言时机如何设计？
