01 Technical Evolution
Part 1

从 LLM 到 CLaw
技术演进与本质

核心叙事线：LLM → Chatbot → Agent（ReAct）→ OpenClaw
每一步跨越意味着什么？对我们（设计者）意味着什么？

01 LLM
01

大模型（LLM）

会说话的大脑

Prompt → LLM → Text
  • 本质上是一个文本生成器
  • 能力：理解语义、生成内容、推理
  • 局限：没有记忆、没有行动力、没有外部连接
  • 形态：单轮问答，用完即走
01 Chatbot
02

Chatbot

能对话了

  • 在 LLM 基础上加入对话历史
  • 多轮对话，有上下文感知
  • 本质仍是被动应答：你问它答，你不问它就不动
  • 形态：ChatGPT、Claude 对话界面

FOR DESIGNERS

交互从搜索框/表单 → 对话框，但仍然是用户发起

01 Agent
03

Agent

会思考、会行动

关键跨越：LLM + Tool Use + ReAct 循环

REACT LOOP

Thought → Action → Observation ↻
  • LLM 决定调用工具来完成任务
  • 角色从"文本生成器"变成"决策者 + 协调者"
  • 模仿人类：想一想 → 做一做 → 看结果 → 再想

FOR DESIGNERS

AI 从"应答工具"变成"能执行任务的助手"
交互从"对话"扩展到"委托"

Agent 住进了电脑

常驻运行 + 通讯连接 + 记忆系统 + 定时心跳

01 Architecture

五大核心组件

01

Gateway

消息路由控制面，连接 20+ 通讯平台。Hub-and-spoke 架构，Channel Adapter 归一化消息格式。

02

Brain

基于 ReAct 循环编排 LLM 调用，支持 Claude、GPT、DeepSeek 等多模型。

03

Memory

双层记忆：短期（daily logs）+ 长期（Markdown 文件）。刻意简洁，无向量数据库。

04

Skills

插件能力系统，YAML frontmatter 定义。社区通过 ClawHub 共享 1700+ 技能。

05

Heartbeat

周期性触发（如每30分钟），无需用户输入。Agent 主动性的关键机制。

01 Agent Loop

Agent Loop — 完整循环

接收消息 → 路由到会话 → 加载上下文+Skills → LLM 推理 (ReAct) → 工具调用 → 流式回复 → 持久化状态

FOR DESIGNERS

这不再是一个"工具"，而是一个有记忆、有行动力、能主动联系你、活在你日常通讯渠道里的数字实体。
交互设计的对象从"界面"变成了"关系"。

02 Memory

2.2 连续交互与
记忆能力

DESIGN QUESTION

如何设计"长期关系型"的交互？
记忆如何影响 Agent 的行为和人格？

短期记忆

长期记忆

02 End-User Dev

2.3 终端用户编程与软件定制

从"我来编程"到"Agent 自己开发"的递进

L1

直接编程

开发者写代码构建定制系统，门槛最高

L2

Coding Agent 对话式开发

用自然语言描述需求，Agent 帮你写代码（如 Claude Code、Cursor）

L3

OpenClaw Runtime 自适应

运行时告诉 Agent 你要什么功能，Agent 自己开发、部署、运行。不再有"开发"和"使用"的边界

02 Challenges

Agent 时代的
设计挑战

以上是 Agent 架构带来的创新机会，以下是设计者需要解决的问题

02 Challenges
Challenge 01

透明性与控制权

OpenClaw 实际上是个黑盒：权限极高，消耗大量 token，后台做了很多用户看不到的事情。

Agent 在做什么？做了多少？做对了没有？
如何设计 Agent 行为的可审计性？

Challenge 02

信任校准

用户如何建立对 Agent 的合理信任？

过度信任 → 依赖 Agent，出错不自知
过度怀疑 → 反复确认，Agent 形同虚设

如何帮助用户形成准确的心智模型？

02 Challenges
Challenge 03

错误恢复与问责

Agent 自主执行了错误操作怎么办？发错邮件、删错文件、回复不当。

如何设计撤销/回滚机制？
出了问题"谁负责"？

Challenge 04

隐私与记忆边界

Agent 记住了太多个人信息怎么办？记忆有时效性，过时的记忆可能导致错误判断。

如何设计记忆衰减机制？
多用户场景中记忆的边界在哪里？

Challenge 05

社交场景中的 Agent

Agent 出现在群聊中时，它是什么角色？是某人的私人助手还是群组的公共助手？

Agent 在多人交互中的身份、权限、发言时机如何设计？
